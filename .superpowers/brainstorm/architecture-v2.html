🏗️ 架构升级：盲操作 → 感知型 GUI 自动化

核心洞察：模拟真人需要"眼睛"，不只是"手"

一、问题：当前 GUI 脚本是"盲的"

v1 盲操作 ui-control.sh

发射键盘指令，祈祷它能工作：

Cmd+L       # 希望 Cascade 打开了
sleep 0.3   # 希望够快
Cmd+V       # 希望输入框有焦点
Enter       # 希望消息发出去了

❌ 不知道屏幕上发生了什么

❌ 弹窗/loading/焦点丢失 → 静默失败

❌ 无法检测 rate limit 弹窗

❌ 无法确认消息是否发送成功

❌ 无法找到正确的 tab

v2 感知型 gui-agent.py

看屏幕 → 理解 → 行动 → 验证：

screenshot()           # 先看
state = analyze(img)   # 理解当前状态
if state.has_dialog:   # 处理意外情况
    dismiss_dialog()
focus_input()          # 精确定位输入框
type_message(msg)      # 输入
verify_sent()          # 确认发送成功

✅ 每一步都有视觉验证

✅ 能处理弹窗、loading、焦点丢失

✅ 能在屏幕上识别 rate limit 文字

✅ 知道消息是否真的发出去了

✅ 能识别并切换 tab

二、感知-行动循环 (Perceive-Act Loop)

真人操作 GUI 的每一步都是：看 → 判断 → 做 → 验

📸
PERCEIVE
截图
🧠
ANALYZE
识别状态
🎯
PLAN
决定动作
⚡
ACT
执行操作
✅
VERIFY
确认结果
stateDiagram-v2
    [*] --> Perceive: 开始操作

    Perceive --> Analyze: 截图完成
    Analyze --> Plan: 状态已识别

    state Plan {
        [*] --> CheckState
        CheckState --> HandleDialog: 有弹窗
        CheckState --> HandleLoading: 加载中
        CheckState --> HandleRateLimit: 检测到限速
        CheckState --> ReadyToAct: 正常状态
        HandleDialog --> ReadyToAct: 弹窗已处理
        HandleLoading --> Perceive: 等待 → 重新截图
    }

    Plan --> Act: 动作已确定
    Act --> Verify: 动作已执行
    Verify --> Success: 验证通过
    Verify --> Perceive: 验证失败 → 重试

    Success --> [*]

    note right of HandleRateLimit: 上报 OpenClaw\n不自行决策

三、"发送一条消息"的真实步骤

人类发送一条 Cascade 消息，实际上经历了 7 个微步骤，每一步都可能失败。 
sequenceDiagram
    participant G as GUI Agent
    participant S as Screen
    participant W as Windsurf

    G->>S: ① 截图: 当前屏幕状态
    G->>G: ② 分析: Cascade面板在哪？输入框在哪？
有弹窗吗？正在loading吗？

    alt Cascade 不可见
        G->>W: ③a AppleScript: Cmd+L 打开
        G->>S: ③b 截图验证: Cascade 打开了？
    end

    alt 有弹窗阻挡
        G->>W: ③c 处理弹窗 (ESC / 点击关闭)
        G->>S: ③d 截图验证: 弹窗消失了？
    end

    G->>G: ④ 定位输入框 (OCR/模板匹配)
    G->>W: ⑤ 点击输入框 → 粘贴消息
    G->>S: ⑥ 截图验证: 文字出现在输入框中？
    G->>W: ⑦ 按 Enter 发送

    G->>S: ⑧ 截图验证: 消息出现在对话中？
    G->>G: ⑨ 返回结果: success / retry / error

六、职责矩阵 v2

角色职责不做什么技术栈
Quote 插件 换号、配额查询、状态 API 不操作 GUI、不做决策 TypeScript, bridge.ts HTTP
GUI Agent
gui-agent.py		 感知屏幕状态 → 执行 GUI 操作 → 验证结果 不换号、不做业务决策 Python + screencapture + Vision/OCR + AppleScript
OpenClaw 决策大脑：何时换号、何时重试、何时新 session 不直接操作任何东西 规则引擎 + webhook
Cascade Hook
cascade-hook.py		 解析 Cascade 回复 → 分类事件 → 通知 OpenClaw 不做决策、不做操作 Python, stdin → HTTP POST

七、GUI Agent CLI 接口

关键变化：从 4 个简单命令变为 感知型命令，每个命令内部都有截图→分析→执行→验证循环。
命令内部流程返回
gui-agent.py status 截图 → 分析 → 返回 JSON 状态
(cascade可见? 输入框位置? 有弹窗? loading?)		 {"cascade_visible": true, "input_ready": true, "has_dialog": false}
gui-agent.py send "消息" 截图 → 确认 Cascade 可见 → 定位输入框 → 粘贴 → 发送 → 验证 {"success": true, "verified": true}
gui-agent.py new-tab "消息" 截图 → Cmd+L×2 → 验证新 tab → 粘贴 → 发送 → 验证 {"success": true, "tab": "new"}
gui-agent.py read-state 截图 → OCR Cascade 面板 → 提取最后一条回复文本 {"last_response": "...", "has_error": false}
gui-agent.py dismiss-dialog 截图 → 检测弹窗 → ESC/点击关闭 → 验证消失 {"success": true}
