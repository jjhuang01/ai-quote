# PRD

## Product Goal

把一个混淆后的 VSIX 还原成一个可持续维护的 VS Code 扩展工程，而不是一坨不可读的逆向产物。

## User Value

- 用户可以继续使用 Echo 的核心入口与本地桥接能力。
- 用户可以在当前 IDE 中获得 MCP 配置与反馈规则自动注入。
- 用户可以看到明确的状态、日志与故障信息。
- 工程团队可以在此基础上继续补齐 parity，而不是重新从零取证。

## Success Metrics

1. 扩展能够正常安装、激活、显示侧边栏。
2. 4 个命令均可执行，并产生明确可观察结果。
3. 本地 bridge 正常启动，关键端点可访问。
4. 至少 1 个 IDE MCP 配置文件能被正确写入；其余路径按证据等级实施。
5. 全部验证命令通过。
